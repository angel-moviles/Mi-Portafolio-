Contexto del Proyecto

Problema real: Farmacias tradicionales con control manual de inventario, pérdidas por vencimiento de productos, ventas lentas y nula trazabilidad.

Mi Rol: Arquitecto de software & desarrollador Full Stack (backend Laravel + frontend React/React Native).

Impacto medible: Reducción del 95% de errores de inventario, ventas procesadas en <30 segundos, alertas automáticas de stock crítico.

Arquitectura: API REST consumida por dos frontends (web y móvil), base de datos relacional optimizada, autenticación por roles (Admin, Farmacéutico, Cajero, Auxiliar).

Video demostrativo completo: Flujo de ventas, gestión de inventario, reportes y panel administrativo (Frontend React + Backend Laravel)

Arquitectura y diseño

El sistema sigue una arquitectura de microservicios orientada a API REST, garantizando escalabilidad y mantenibilidad. El backend está construido con Laravel 11.La autenticación se maneja con Laravel Sanctum (tokens JWT) y control de acceso basado en roles. Por otro lado, los clientes incluyen:

  • Web Dashboard (React + Vite): Panel administrativo con gráficos interactivos (Recharts), gestión completa de productos, ventas, proveedores y usuarios.
  • App Móvil (React Native + Expo): Punto de venta con escáner de código de barras, recepción de notificaciones push, generación de tickets PDF y modo offline con sincronización.

Resultados y logros

FarmaPOS fue concebido como un proyecto universitario que evolucionó a un producto funcional demostrando mi capacidad y la de mi equipo para integrar soluciones completas. Entre los logros destacan:

  • Reducción del tiempo de venta de 3 minutos a menos de 30 segundos.
  • Control total de inventario con alertas automáticas de caducidad y bajo stock.
  • Seguridad robusta con roles diferenciados y registro de auditoría por cada transacción.
  • Interfaz moderna y responsiva tanto en web como en dispositivos móviles.
  • Documentación completa de API (Postman) y despliegue mediante contenedores Docker.
